# 贡献指南 (Contributing Guide)

感谢您关注本项目！我们非常欢迎并期待您的贡献。以下是参与本项目建设的基本规范：

## 开发流程
1. **提交 Issue**：在开始修改前，请先提交或认领一个相关的 Issue，阐明修改动机。
2. **拉取分支**：请勿直接修改 `main` 分支。请拉取格式为 `feat-xxx` 或 `fix-xxx` 的本地工作分支进行开发。
3. **编写代码**：请遵守团队制定的全局 CSS 变量规范，确保静态页面在不同设备和主题下的风格统一。
4. **提交 Pull Request (PR)**：修改完成后，向主分支提交 PR。请在描述中关联对应的 Issue（如 `Closes #1`）。
5. **代码审查**：所有 PR 必须通过静态检查（CI）并获得至少 1 名项目成员的审核通过（Approved）后方可合入主干。

再次感谢您的支持！
